# Periodic Table Database System 

A structured database management project designed to organize, store, and retrieve chemical element data efficiently. This project demonstrates foundational skills in Database Schema Design and SQL Query Optimization.

##  Project Overview
The Periodic Table Database System is a back-end focused project that manages data for all 118 chemical elements. It provides a systematic way to access atomic properties, classifications, and trends within the periodic table through relational database concepts.

##  Key Features
- Relational Data Mapping: Elements are organized into tables with relationships between atomic numbers, symbols, and chemical groups.
- Efficient Search: Implemented SQL queries to quickly find elements by name, symbol, or atomic number.
- Categorization: Ability to filter elements based on properties such as Metals, Non-metals, Noble Gases, etc.
- Data Integrity: Ensured precise data entry for atomic mass, boiling points, and electronegativity.

##  Tech Stack
- Database: MySQL / SQL Server
- Language: SQL
- Concepts: Data Normalization, Entity-Relationship Modeling (ERM)

##  Database Schema
The database includes the following key attributes for each element:
- Atomic Number (Primary Key)
- Element Name & Symbol
- Atomic Mass
- Group & Period
- Electron Configuration
- Standard State (Solid/Liquid/Gas)

##  How to Use
1. Clone the repository.
2. Import the .sql file into your local MySQL environment.
3. Run the provided queries to view or filter the element data.
